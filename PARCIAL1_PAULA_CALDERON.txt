Sección 3 Análisis y Rendimiento

Pregunta A: Elección de Estructuras de Datos

Para almacenar la información de los estudiantes por programa utilicé un diccionario, donde cada programa tiene asociados varios contadores como hombres, mujeres, total de estudiantes y la suma de los promedios. Esto se puede ver en la función donde se crean los datos del semestre.

Usar un diccionario resulta más eficiente y ordenado que manejar muchas listas o variables independientes. Si se utilizaran varias listas separadas (por ejemplo una para hombres, otra para mujeres, otra para promedios), sería más difícil mantener la relación entre los datos. En cambio, con el diccionario toda la información del programa queda agrupada en un solo lugar.

Además, si en el futuro se agregaran más programas o más datos estadísticos, el diccionario permite escalar el sistema fácilmente sin modificar demasiado la estructura del código.



Pregunta B: Optimización de Ciclos y Complejidad

En el proceso de finalización de semestre utilicé un ciclo for para registrar cada estudiante y dentro de ese ciclo se solicitan las 5 notas mediante otro ciclo for. De esta forma se controla que siempre se ingresen exactamente cinco notas por alumno.

Primero se recorren los estudiantes y, para cada uno, se leen las cinco notas y se calcula el promedio individual usando: promedio = sum(notas) / CANTIDAD_NOTAS

Luego ese promedio se acumula directamente en el diccionario del programa en la variable suma_promedios.

De esta manera no es necesario recorrer nuevamente una lista de promedios individuales para calcular el promedio del programa. Simplemente se divide la suma de promedios acumulada entre el total de estudiantes del programa cuando se genera el reporte.



Pregunta C: Mutabilidad y Seguridad de Datos

En el caso de datos que no cambian, como los programas disponibles, utilicé tuplas, por ejemplo en la constante PROGRAMAS.

La ventaja de usar tuplas en lugar de listas es que las tuplas son inmutables, es decir, sus valores no pueden modificarse después de ser creados. Esto ayuda a proteger la información y evita que accidentalmente se cambien datos que deberían permanecer constantes durante la ejecución del programa.

Esto ayuda a mantener la seguridad de los datos y evita cambios accidentales en valores que deberían permanecer constantes.
